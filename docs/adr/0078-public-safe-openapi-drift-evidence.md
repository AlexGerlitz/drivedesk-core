# ADR-0078: Public-Safe OpenAPI Drift Evidence

## Status

Accepted

## Context

DriveDesk publishes a generated OpenAPI contract in the public engineering
surface. That contract drives the public demo API, generated SDK examples,
GitHub Pages links, and several verification scripts.

If application routes or schemas change without regenerating `docs/openapi.json`,
the public repository can present a stale API contract even while individual
tests still pass.

## Decision

Add a public-safe OpenAPI drift evidence layer:

- `docs/public/OPENAPI_DRIFT.md` explains the contract;
- `docs/public/evidence/openapi-drift.sanitized.json` records the sanitized
  evidence snapshot;
- `scripts/check_public_openapi_drift.sh` compares committed `docs/openapi.json`
  with the schema generated by `drivedesk_api.main:app.openapi()`;
- the checker validates the generated SDK manifest and public demo markers;
- the checker is included in public smoke, private smoke, export, and release
  gate validation.

## Consequences

- API changes must update the committed public OpenAPI artifact.
- SDK examples stay tied to the same public OpenAPI source.
- Public demo health can be traced back to the generated API contract.
- No private endpoints, credentials, hostnames, raw logs, request bodies, or
  production data are exposed.
